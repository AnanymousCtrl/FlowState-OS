# FlowState OS

FlowState OS is a browser-based productivity workspace that brings coding,
documentation, notes, and tasks into one React application. It includes an
AI-assisted code editor, a rich-text document editor, local note and task
management, and an experimental real-time collaborative editor powered by Yjs.

## Features

- **AI code workspace** - Write code in Monaco, choose from 15 languages, and
  ask Gemini to review, fix, explain, or predict its output.
- **JavaScript sandbox** - Run JavaScript in an isolated browser iframe and
  inspect console output and runtime errors.
- **Document studio** - Create formatted content with TipTap, including
  headings, lists, alignment, links, images, inline code, and undo/redo.
- **Notes and tasks** - Create, edit, search, tag, archive, and restore notes;
  manage and reorder tasks alongside them.
- **Local persistence** - Notes and tasks are stored in browser `localStorage`.
- **Collaborative editor** - An experimental Monaco editor synchronizes text
  and user presence through Yjs, IndexedDB, and a local WebSocket server.

## Getting Started

1. Clone the repository:

   ```bash
   git clone https://github.com/AnanymousCtrl/FlowState-OS.git
   cd FlowState-OS
   ```

2. Install the frontend dependencies:

   ```bash
   cd frontend
   npm install
   ```

3. Start the development server:

   ```bash
   npm run dev
   ```

4. Open the URL printed by Vite, normally `http://localhost:5173`.

The main application does not require the WebSocket backend. Start the backend
only when working on the experimental collaborative editor.

## Collaborative Editor

The collaborative editor connects to `ws://localhost:1234` and stores a local
copy of each Yjs document in IndexedDB.

In a second terminal:

```bash
cd backend/y-websocket-server
npm install
node server.js
```

Then visit `http://localhost:5173/codeEditor`.

For deployment, update the WebSocket URL in
`frontend/src/code/CodeEditor.jsx` to use the deployed server and `wss://` when
the site is served over HTTPS.

## Data Storage

- Notes, tasks, and their last-sync timestamp are saved in browser
  `localStorage`.
- Collaborative documents are cached in browser IndexedDB.
- Clearing browser site data removes locally stored content.
- The WebSocket server relays collaborative changes but does not currently
  provide durable server-side persistence.

## Important Development Notes

- The Gemini API key is currently initialized in
  `frontend/src/code/Code.jsx`. Before publishing or deploying the project,
  remove any hard-coded key, rotate exposed credentials, and proxy AI requests
  through a secure backend or another server-side secret store.
- JavaScript can be executed locally in the browser sandbox. For other
  languages, the displayed execution result is generated by the AI model and
  is not produced by a language runtime.
- The document editor content is not currently persisted after a page refresh.
- The collaborative route currently needs proper room and display-name wiring
  before it is production-ready.
- Automated tests have not yet been added.
- The production build succeeds, but `npm run lint` currently reports existing
  ESLint violations that still need to be resolved.
